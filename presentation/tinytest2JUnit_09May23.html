3 Ways to Run Tests in Tinytest

tinytest::test_package("mypkg", testdir = "inst/tinytest") tinytest::test_all(pkgdir = "mypkg") tinytest::run_test_dir(system.file("mypkg", package="tinytest2JUnit"))

  • test_package: test a package during R CMD check -> causes pipeline to fail on fail of any test
  • test_all: runs tests interactively -> causes pipeline to fail on fail of any test
  • run_test_dir is the only funcion that stores test results, without exiting Jenkins pipeline
